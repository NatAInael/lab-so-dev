# FASE 2 — Assembly (gcc -S)
# Cole aqui 5 a 10 linhas de soma.s que você considera mais
# interessantes e explique o que cada uma faz.

Trecho de soma.s:
    .file   "soma.c"
    .text
    .globl  main
    .type   main, @function
main:
    push    rbp
    mov     rbp, rsp
    mov     edi, 2
    mov     esi, 3
    call    printf
    mov     eax, 0
    pop     rbp
    ret

Explicação:
- `.globl main` → define a função main como global, visível ao linker.
- `push rbp` / `mov rbp, rsp` → cria o stack frame da função.
- `mov edi, 2` / `mov esi, 3` → passa argumentos para printf (convenção x86_64).
- `call printf` → chamada de função; transição para o código do printf.
- `mov eax, 0` → valor de retorno da função main.
- `pop rbp` / `ret` → finaliza a função, restaurando o stack frame.

- `printf` → símbolo indefinido (U = undefined), será resolvido pelo linker
  usando a biblioteca padrão do C.

# QUESTÕES DE SÍNTESE

# 1. O nm mostrou printf como "U" (undefined). Em qual slide da
#    apresentação esse mesmo problema apareceu? O que o ligador
#    faz com esse símbolo?

Resposta 1:
O problema aparece nos slides sobre **ligação de símbolos externos** (linking). O ligador procura a implementação de `printf` nas bibliotecas externas (libc), combina o código objeto com essas bibliotecas e resolve os símbolos indefinidos para gerar o executável final.
